# Content Aware Widget Arbitration - Developer SDK Integration Guide

## Overview

This SDK allows widget-based Android applications to determine whether a widget should refresh or suppress content during lock/unlock screen events. It is powered by an AI-based engagement prediction model and customizable policy overrides.

## Modules Included

- `WidgetArbitrationManager` ‚Äî Orchestrates arbitration logic
- `EngagementScorer` ‚Äî Simulated AI scoring module
- `OverridePolicyHandler` ‚Äî Checks app/carrier-based override rules
- `WidgetRefreshDecision` ‚Äî Data class for arbitration result

## Integration Steps

### 1. Broadcast Hook

Register a `BroadcastReceiver` or extend `AppWidgetProvider` to receive:
```kotlin
IntentFilter().apply {
    addAction(AppWidgetManager.ACTION_APPWIDGET_UPDATE)
}
```

### 2. Provide Widget Metadata

Create a metadata map with:
- `type`: content type (e.g., "news", "weather")
- `lastRefresh`: timestamp
- `appPackage`: context.packageName

### 3. Call Arbitration Engine

Use:
```kotlin
val decision = WidgetArbitrationManager.shouldRefreshWidget(widgetId, metadata, context.packageName)
```

### 4. React to Decision

If `decision.shouldRefresh == true`, perform your widget content update. Otherwise, skip refresh.
